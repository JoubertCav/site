Functions, Images, and Preimages

A function is a deterministic arrow from each point of a domain to exactly one point of a codomain. Images move sets forward. Preimages pull predicates backward.

f−1(B) = { x∈X : f(x)∈B }

Interpretation

What to notice

The forward image only sees points that are actually hit by arrows. The inverse image is different: it pulls a subset of the codomain back to every domain point whose arrow lands there.

Exam-style summary

minimal checklist
  • Function: each x has one and only one image.
  • Injective: no two distinct domain points share the same image.
  • Surjective: every codomain point is hit.
  • Bijective: injective + surjective; then an inverse function exists.
  • Image: pushes A⊆X forward to f(A)⊆Y.
  • Preimage: pulls B⊆Y backward to f−1(B)⊆X; always defined for any function.
  • Composition: first f, then g; order matters.
  • Restriction: same rule on smaller domain. Extension: a larger-domain function agreeing on the old domain.
Finite diagrams reveal the general logic: functions are graphs with exactly one output per input; images and preimages are set transformations.
